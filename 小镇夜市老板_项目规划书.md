# 《小镇夜市老板》项目规划书 v0.1

## 1. 项目定位

### 1.1 一句话概念

玩家从一个小推车开始经营夜市摊，通过做菜、接单、升级摊位、解锁街区，逐步把冷清小巷经营成热闹夜市街。

### 1.2 推荐平台

- 首发平台：微信小游戏或抖音小游戏
- 首发商业化：IAA 广告变现，不做充值、不做内购、不做提现
- 开发引擎：Cocos Creator 3.x + TypeScript
- 目标包体：首包 15MB 以内，远程资源按章节加载
- 单局时长：60-150 秒
- 单次会话：5-12 分钟
- 目标用户：三四线及以下城市用户、轻度休闲用户、生活题材偏好用户、短视频导流用户

### 1.3 设计原则

- 题材下沉：烧烤摊、小吃街、学校门口、工厂夜班、庙会大集等生活场景。
- 操作简单：以点击、拖拽、排队、收菜为主，不做复杂手势。
- 难度可控：前 30 关只引入少量菜品和设备，避免早期流失。
- 内容可量产：菜品、顾客、关卡、摊位皮肤都通过配置表驱动。
- 合规安全：无充值、无红包提现、无博彩感抽奖、无强社交、无敏感历史/政治/宗教内容。

## 2. 核心玩法

### 2.1 核心循环

1. 关卡开始，顾客排队出现。
2. 顾客提出订单，例如“烤冷面 + 奶茶”。
3. 玩家点击/拖拽食材到对应设备。
4. 设备经过短时间烹饪后产出菜品。
5. 玩家把菜品交给顾客，获得金币和口碑。
6. 关卡结束，根据金币、服务人数、连击获得星级。
7. 关卡外使用金币升级设备、解锁菜品、装饰摊位。
8. 进入下一关或新街区。

### 2.2 MVP 只做三类操作

- 点击备菜：点击食材生成半成品。
- 点击烹饪：把半成品放入烤炉/铁板/饮品台。
- 点击上菜：菜品完成后点击顾客交付。

第一版不做自由摆摊地图、不做复杂路径、不做实时多人、不做充值商城。

### 2.3 失败与复活

关卡不做强失败惩罚。时间结束后如果没有达成 1 星目标，进入结算失败页。

可选激励广告：

- 失败后看广告延长 15 秒。
- 成功后看广告获得本局金币 x2。
- 每日首次登录看广告领取装饰材料。

## 4. 关卡类型

### 4.1 标准营业关

目标：在限定时间内赚够金币。

适用：大多数普通关卡。

参数特征：

- 客流稳定。
- 菜品池为当前章节主菜品。
- 失败压力较低。

目标金币公式：

```text
expectedRevenue(L) = sum(orderCount(dish, L) * dishPrice(dish, L)) * successRate(L)
goalCoin_1star(L) = roundTo10(expectedRevenue(L) * 0.65)
goalCoin_2star(L) = roundTo10(expectedRevenue(L) * 0.82)
goalCoin_3star(L) = roundTo10(expectedRevenue(L) * 0.96)
```

### 4.2 高峰客流关

目标：应对短时间集中客流。

适用：每章第 5、10、15 关附近。

参数特征：

- 总顾客数 +20%。
- 顾客生成分成 2-3 个高峰波次。
- 顾客耐心 -10%。
- 奖励金币 +10%。

公式：

```text
customerCount = baseCustomerCount(L) * 1.20
patienceMultiplier = 0.90
rewardMultiplier = 1.10
waveCount = 2 + floor(chapter(L) / 3)
```

### 4.3 指定菜品冲刺关

目标：在限定时间内卖出指定菜品。

适用：新菜品解锁后的第 2-3 关。

参数特征：

- 指定菜品订单权重提高到 45%-60%。
- 用于教学玩家熟悉新设备或新菜。

公式：

```text
targetDishWeight = clamp(0.45 + 0.03 * chapter(L), 0.45, 0.60)
targetDishGoal = ceil(baseCustomerCount(L) * targetDishWeight * 0.75)
```

### 4.4 连击挑战关

目标：连续满意服务一定数量的顾客。

适用：每章中后段。

参数特征：

- 顾客耐心略高。
- 超时或上错菜会断连。
- 适合引导升级设备。

公式：

```text
comboGoal = ceil(4 + chapter(L) * 1.5 + localIndex(L) * 0.15)
patienceMultiplier = 1.10
wrongServePenalty = breakCombo
```

### 4.5 大份订单关

目标：处理多菜组合订单。

适用：第 3 章开始。

参数特征：

- 30%-50% 的顾客会点 2 个菜。
- 收入高，但设备占用更紧张。

公式：

```text
multiItemOrderRate = clamp(0.20 + 0.06 * chapter(L), 0.20, 0.55)
avgItemsPerOrder = 1 + multiItemOrderRate
goalCoin = standardGoalCoin(L) * (1 + 0.12 * multiItemOrderRate)
```

### 4.6 随机事件关

目标：在小事件干扰下完成营业。

适用：第 4 章以后，每章 2-3 关。

可选事件：

- 下小雨：顾客数 -10%，单价 +15%。
- 网红探店：前 30 秒客流 +30%。
- 设备老化：一个设备制作时间 +20%。
- 赶集日：游客比例 +25%，组合订单 +15%。

公式：

```text
eventIntensity = clamp(0.08 + 0.02 * chapter(L), 0.08, 0.20)
eventRewardBonus = eventIntensity * 1.5
```

### 4.7 Boss 营业日

目标：每章第 20 关，用该章节所有机制做一次综合考试。

参数特征：

- 时间 +15%。
- 顾客数 +25%。
- 订单复杂度 +1 档。
- 通关后解锁新街区。

公式：

```text
duration = baseDuration(L) * 1.15
customerCount = baseCustomerCount(L) * 1.25
complexityBonus = 1
goalCoin = standardGoalCoin(L) * 1.20
```

## 5. 关卡参数公式

### 5.1 基础索引

```text
L = 关卡编号，范围 1-100
C = chapter(L) = floor((L - 1) / 20) + 1
I = localIndex(L) = ((L - 1) % 20) + 1
D = difficulty(L) = 1 + 0.045 * (L - 1) + 0.12 * (C - 1)
```

### 5.2 关卡时长

```text
baseDuration(L) = clamp(70 + 1.1 * L + 4 * C, 75, 150)
```

建议：

- 1-10 关：75-90 秒。
- 11-40 关：90-120 秒。
- 41-100 关：110-150 秒。

### 5.3 顾客数量

```text
baseCustomerCount(L) = ceil(6 + 0.75 * L + 0.018 * L^2)
customerCount(L, typeModifier) = round(baseCustomerCount(L) * typeModifier)
```

其中 typeModifier：

- 标准营业关：1.00
- 高峰客流关：1.20
- 指定菜品关：0.95
- 连击挑战关：0.90
- 大份订单关：0.85
- 随机事件关：0.90-1.25
- Boss 关：1.25

### 5.4 顾客生成间隔

```text
avgSpawnInterval = duration / customerCount
spawnInterval = avgSpawnInterval * random(0.75, 1.25)
```

高峰关使用波次：

```text
waveStartTimes = [0.15 * duration, 0.48 * duration, 0.75 * duration]
waveDuration = duration * 0.15
waveSpawnInterval = avgSpawnInterval * 0.55
```

### 5.5 顾客耐心

```text
customerPatience = basePatience(customerType)
                 * stagePatienceMultiplier(levelType)
                 * (1 - 0.004 * L)
                 * (1 - 0.04 * (orderItemCount - 1))
```

保底：

```text
customerPatience = clamp(customerPatience, 14s, 42s)
```

### 5.6 菜品价格

```text
dishPrice(dish, L) = roundTo1(
  basePrice(dish)
  * chapterPriceMultiplier(C)
  * dishLevelMultiplier(dishLevel)
)

chapterPriceMultiplier(C) = 1 + 0.18 * (C - 1)
dishLevelMultiplier(dishLevel) = 1 + 0.08 * (dishLevel - 1)
```

### 5.7 制作时间

```text
cookTime(dish) = baseCookTime(dish)
               * (1 - equipmentSpeedBonus(equipmentLevel))
               * eventCookModifier

equipmentSpeedBonus(level) = clamp(0.05 * (level - 1), 0, 0.35)
```

### 5.8 订单权重

每个关卡从已解锁菜品里抽取 2-5 个菜品进入订单池。

```text
unlockedDishCount(L) = count(dish.unlockLevel <= L)
dishPoolSize(L) = clamp(2 + floor(L / 15), 2, 5)
```

普通菜品权重：

```text
weight(dish) = 1 / dishComplexity(dish)
```

新菜保护：

```text
if L - unlockLevel(dish) <= 3:
  weight(dish) = weight(dish) * 2.5
```

指定菜品冲刺关：

```text
targetDishWeight = 0.45-0.60
otherDishesShare = 1 - targetDishWeight
```

### 5.9 星级评分

评分由金币、满意顾客、连击构成。

```text
score = coinEarned
      + satisfiedCustomerCount * 5
      + maxCombo * 8
      - angryLeaveCount * 10
      - wrongServeCount * 8
```

星级线：

```text
expectedScore(L) = goalCoin_1star(L) + customerCount(L) * 5 + expectedCombo(L) * 8
star1 = expectedScore * 0.70
star2 = expectedScore * 0.86
star3 = expectedScore * 1.00
```

### 5.10 升级成本

设备升级成本：

```text
equipmentUpgradeCost(equipment, level, C) =
roundTo10(baseEquipmentCost(equipment)
  * level^1.65
  * (1 + 0.35 * (C - 1)))
```

菜品升级成本：

```text
dishUpgradeCost(dish, level, C) =
roundTo10(baseDishCost(dish)
  * level^1.55
  * (1 + 0.25 * (C - 1)))
```

升级收益：

- 菜品升级：每级售价 +8%。
- 设备升级：每级制作时间 -5%，最多 -35%。
- 摊位升级：同时开工位 +1 或排队位 +1，属于关键升级。

### 5.11 经济回收周期

让玩家每 2-4 关能买一次小升级，每 8-12 关能买一次关键升级。

```text
avgCoinPerLevel(L) = expectedRevenue(L) * 0.82
minorUpgradeCost(L) = avgCoinPerLevel(L) * random(1.8, 3.2)
majorUpgradeCost(L) = avgCoinPerLevel(L) * random(7.0, 10.0)
```

## 8. 系统模块拆分

### 8.1 核心模块

- GameLoop：关卡计时、状态切换、暂停、结算。
- OrderSystem：生成顾客、生成订单、订单超时、订单完成。
- CookingSystem：设备队列、烹饪计时、菜品产出。
- EconomySystem：金币、星级、奖励结算、升级消耗。
- LevelConfigLoader：读取关卡配置，按公式生成缺省参数。
- UpgradeSystem：菜品升级、设备升级、摊位升级。
- AdSystem：激励广告、插屏广告、广告冷却。
- SaveSystem：本地存档，云存档作为后续可选。
- AnalyticsSystem：关卡开始、失败、完成、广告展示、广告完成等埋点。

### 8.2 场景结构

- LoadingScene：资源加载。
- HomeScene：摊位主界面、升级入口、关卡入口。
- GameScene：核心营业玩法。
- ResultScene：结算、广告翻倍、下一关。
- UpgradeScene：设备、菜品、装饰升级。

## 9. 美术与表现

### 9.1 风格

- 2D 俯视斜角或正面摊位视角。
- 色彩温暖但不廉价，夜市灯牌、塑料凳、铁板油光、蒸汽、烟火气。

- 菜品图标：12 个。
- 顾客立绘：6 个基础顾客，每个 2 个表情。
- UI：金币、星星、计时器、订单气泡、升级按钮、广告按钮。

## 10. 广告变现设计

### 10.1 激励广告点位

| 点位 | 触发 | 奖励 | 限制 |
|---|---|---|---|
| 结算翻倍 | 通关后 | 本局金币 x2 | 每关 1 次 |
| 失败续时 | 失败后 | +15 秒 | 每关 1 次 |
| 免费加速 | 关卡外 | 设备加速 10 分钟 | 每日 3 次 |
| 每日补贴 | 首页 | 固定金币 | 每日 1 次 |
| 装饰试用 | 换装页 | 试用装饰 1 局 | 每个装饰 1 次 |

### 10.2 插屏广告

谨慎使用。

建议规则：

```text
if completedLevelCount >= 5
and levelsSinceLastInterstitial >= 3
and sessionTime >= 6min:
  showInterstitial()
```

### 10.3 不做的变现

- 不做充值。
- 不做月卡。
- 不做付费去广告。
- 不做红包提现。
- 不做概率抽奖售卖。

## 11. 数据埋点

### 11.1 核心埋点

- level_start：关卡开始。
- level_complete：关卡成功。
- level_fail：关卡失败。
- ad_show：广告展示。
- ad_complete：广告看完。
- ad_skip/fail：广告未完成。
- upgrade_buy：购买升级。
- dish_unlock：解锁菜品。
- session_start/session_end：会话时长。

### 11.2 关键指标

- D1 留存：目标 25%-35%。
- D3 留存：目标 10%-18%。
- 单局完成率：前 10 关 >85%，11-30 关 >70%。
- 激励广告完成率：目标 >55%。
- 人均广告展示：首版 3-6 次/日。
- 前 5 分钟流失点：重点看教学关、升级页、首次失败。

## 14. 风险与规避

### 14.1 玩法过累

风险：时间管理游戏容易手忙脚乱，轻度用户流失。

规避：

- 前 10 关只允许 1-2 个菜品同时出现。
- 顾客耐心保底 20 秒以上。
- 第一次失败给免费续时，不强制广告。

### 14.2 题材不够吸量

风险：夜市经营不如夸张冲突类短视频吸睛。

规避：

- 素材重点展示“从破摊到爆火夜市”的变化。
- 关卡中做高峰排队和金币爆发反馈。
- 装饰升级要有明显视觉变化。

### 14.3 广告过重

风险：早期插屏太多影响留存。

规避：

- 前 5 关不出插屏。
- 激励广告由玩家主动点击。
- 失败续时首日首次免费。

### 14.4 版号与平台资质

风险：如果后续想做充值，必须准备版号和出版合作，周期长。

规避：

- 首发只做 IAA。
- 不做内购、不做付费解锁。
- 配置和 UI 文案避免“充值”“钻石”“礼包”等付费暗示。

## 15. 后续扩展方向

- 摊位装饰：灯牌、桌椅、招牌、锅具皮肤。
- 图鉴收集：地方小吃图鉴、顾客留言。
- 街区解锁：夜市街、河边摊、庙会、景区。
- 限时活动：周末赶集、夏日冰饮节、开学季。
- 轻社交：好友分数榜，不做聊天和交易。
- 短视频素材：爆单挑战、差一秒出餐、破摊升级成夜市街。
